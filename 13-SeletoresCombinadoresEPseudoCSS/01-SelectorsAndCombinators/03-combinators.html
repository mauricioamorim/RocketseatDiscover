Combunators

Combinadores, pois eles trabalham para buscar e combinar seletores a fim de aplicar uma estilização

Descendant combinator

  • Identificado por um espaço   entre os seletores
  • Busca um elemento dentro de outro

Child combinator

  • Identificado pelo sinal > entre dois seletores
  • Seleciona somente o elemento que é filho direto do elemento referenciado
  • Elementos depois do filho direto serão desconsiderados
    • FIca sem padding-bottom css pois o descendent pega só o filho direto
    • FIca sem padding-bottom css pois o descendent pega só o filho direto

Adjacent sibling combinator

  • Identificado pelo sinal + entre dois seletores
  • Seleciona somente o elemento do lado direto que é irmão direto na hierarquia do elemento referenciado
  • sem CSS por não ser o primeiro elemento ao irmão do h2

General sibling combinator

  • Identificado pelo sinal ~ entre dois seletores
  • Seleciona todos os elementos irmãos do elemento referenciado
  • css é aplicado nesse elemento tambem

Utilizando combinadores

Dica

  • Seletores muito específicos tendem a causar dificuldades no reuso das regras de estilização dos elementos
  • Muitas vezes, um simples uso de classes, torna o trabalho mais eficiente
