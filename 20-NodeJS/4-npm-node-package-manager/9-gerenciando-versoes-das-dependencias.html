Gerenciando versões das dependências

Aprendendo a ler o número de uma versão

"dependencies": { "cfonts": "^2.9.1"// ^major.minor.patch }
*, ^, ~
(*)Quando vem no começo, ao rodar "npm upgrade" ele vai atualizar o Major, Minor e Patch (^)Quando vem no começo, ao rodar "npm upgrade" ele vai atualizar o Major e Patch (~)Quando vem no começo, ao rodar "npm upgrade" ele vai atualizar somente o Patch Quando não vem nada no começo, ele nunca atualiza.
Major (2)
Versão principal da dependência, só é alterado quando existe uma nova versão. Se atualizar ou regredir pode ser que a dependência para de funcionar no projeto? Quando é dado um "npm upgrade" ele NÂO atualiza essa versão.
Minor (9)
Quando a mesma versão tem uma melhoria, esse valor é atualizado
Patch (1)
Quando a versão arruma um pequeno bug, esse valor é atulaizado

npm outdated

Esse comando mosta detalhes sobre a versão das dependências (current, wanted, latest e etc)

current
o número da versão atual (exemplo: 1.7.2)
wanted
o número da ultima atualização da mesma versão atual (exemplo: 1.7.5)
latest
a ultíma versão da dependência (exemplo: 2.29.1)

npm upgrade

Esse comando faz atualizar todas as dependências do projeto mantendo o "major" igual

caso vc queria atulizar uma unica dependência vc pode digitar npm upgrade

@latest

Caso sua versão no package.json não esteja com * e vc queria atualizar a dependência para a versão(Major, Minor e Patch) mais atual, Sera necessário rodar o comando de instalção de dependência com o "@latest" no final. Exemplo: npm install cfonts@latest
