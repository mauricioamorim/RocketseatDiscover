process.stdin.on()

Evento que capta os dados de que entram assincronimamente

On é o evento de fica ouvindo quando entra dados

O primeiro parametro significa q quando tiver dados ele vai ou vir uma função

O segundo parametro é a função chamada(callback) quando o evento on "dados" é disparado(após apertar a tecla enter)

Quando criamos o listner de evento, ele tira a função callback da lista de processos (worker threads) e joga para a lista de evento (event queue). Ai o event loop fica observando se o Enter é precionado. Se sim, ele joga a execução da função para a lista de processos (worker threads).
